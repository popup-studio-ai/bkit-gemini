## 5. Key Decisions

### D1 lib/pdca/status.js v3 schema (활성 v2 stale 정정)
hooks/scripts/utils/pdca-state-updater.js + hooks/scripts/before-tool-selection.js + hooks/scripts/before-model.js 모두 v3 스키마(`features` object map + `activeFeatures` string array)를 사용 중이었으나 `lib/pdca/status.js`는 v2 (`activeFeatures` object map). 라이브러리가 hook의 실제 사용 패턴을 따라야 → v3로 truth update. 추가로 8개 누락 wrapper (`addActiveFeature`/`removeActiveFeature`/`getActiveFeatures`/`switchFeatureContext`/`completePdcaFeature`/`addPdcaHistory`/`extractFeatureFromContext`/`initPdcaStatusIfNotExists`) 모두 복구.

### D2 ENTER/EXIT_PLAN_MODE getReadOnlyTools에서 제외
mode change side-effect 있는 도구는 진짜 read-only가 아님. TOOL_ANNOTATIONS의 `readOnlyHint: false`와 정합성 유지.

### D3 명시 skip + 사유 주석 (deprecated cluster)
SKILL-93-36~40 (5), TC109-06~08 (3), COMP-01/02/03/06/07 (5) — 13건은 underlying 모듈/함수가 Gemini CLI native system 또는 S3 agent-dispatch로 대체됨. 명시 skip으로 처리하되 각 case에 사유 주석 + 재방문 조건 명시. "임시방편"이 아닌 정확한 분류.

### D4 legacy `.pdca-status.json` 경로 호환
savePdcaStatus가 paths.pdcaStatus(`.bkit/state/pdca-status.json`) 또는 legacy `.pdca-status.json` 둘 다 지원. legacy 파일이 이미 있으면 그 위치에 저장 (강제 migration 금지) — 사용자 환경 무손상.

## 10. Lessons Learned

1. **라이브러리 vs 실 사용 패턴 mismatch는 cascade fail** — lib/pdca/status.js가 v2 스키마였으나 hooks는 v3 사용. 35 PDCA test가 한 모듈 fix로 28 해결.
2. **Deprecated 명시 skip은 "임시방편"이 아닌 정확한 분류** — Gemini CLI native skill로 위임된 skill-orchestrator는 부활 의미 없음. 명시 skip + 사유 주석으로 future revisit 조건 명확.
3. **Truth-update 패턴의 누적 효과** — version sync(5건), file count(2건), model name(4건) 등 단순 truth-update가 +11. 동적 회귀(BT-01~03/08/11)로 미래 drift 차단.
4. **Wave 단위 격리 commit이 회귀 발견에 유리** — Wave 5 (PDCA 대규모 fix)에서 collateral 14건 (HOOK/E2E/TC39) 발생. 격리됐다면 각 wave 후 측정으로 즉시 확인.
