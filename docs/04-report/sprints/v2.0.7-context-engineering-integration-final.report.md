## 5. Key Design Decisions

### D1 Deterministic extractor + injection point
compactor의 default extractor는 heuristic regex (한글 `결정` + ASCII `D1/TODO/file:line`). LLM-backed extractor는 `setExtractor(fn)` 으로 future-replaceable. 현재는 LLM 비용 없이 동작 + CP-13에서 injection 검증.

### D2 Env toggles는 토글 (default-on)
`BKIT_COMPACTION` / `BKIT_TOOL_RESULT_CLEARING`은 default-on, opt-out 방식. case-insensitive (`false`/`0`/`off`) 모두 인식. CP-05, TR-04에서 검증.

### D3 importFromExtractionPatch는 save() 호출 안 함
caller 책임으로 persist 분리. 21 agent file 무손상 보장 (MX-09). dry-run import + 검증 후 명시적 save() 가능.

### D4 sub-agent isolation은 directive metadata만
Gemini CLI 동기 hook 제약으로 실 isolation enforce는 model side. wrapAsIsolated은 `context.isolated: true` + `isolationPolicy: 'context-fork'` + `outputContract` 명시. 미래 async hook (v0.44.0+) 시점에 직접 fork enforce로 upgrade 가능.

## 7. Lessons Learned

1. **한글 word boundary** — JS regex `\b`는 ASCII-only. `결정`/`미해결` 같은 한글 토큰은 boundary 없이 직접 매칭 필요. CP-06 검증으로 발견 후 즉시 fix.
2. **TC110-02 drift cascade** — 새 모듈 추가마다 lib/core/ 카운트 갱신 필요 (16→18). 미래에는 동적 count 또는 정합성 회귀로 자동화 검토 (S7+S2에서 도입한 SP-* 패턴과 유사).
3. **Default-on toggle은 silent disable 방지** — `BKIT_COMPACTION`을 default-off 했으면 사용자가 의식적으로 활성화 필요. default-on + 명시적 opt-out이 안전한 선택.
4. **caller 책임 분리** — Memory import가 save() 자동 호출하지 않음으로써 dry-run import + verify + commit 패턴 가능. 21 file 무손상 보장.
