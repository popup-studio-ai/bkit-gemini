## 6. Lessons Learned

### 6.1 What Went Well (Keep)
- **Wave 0 RCA 우선**: 가설 (R7-G allow silent ignore)을 reproducer로 입증/반증 → 후속 작업 재설계 신속
- **TDD Red→Green 사이클**: trust-score, cmd-parser, before-phase 모두 unit test 우선 작성 → bug 조기 발견 (rejection always-record)
- **Dual normalize (cmd-parser)**: BS-applied + control-stripped 양면 검증 → false-negative 회피 (CP-25)
- **부트스트랩 paradox 명시**: S7 자체 L2 진행을 디자인에 명문화 → 작업 흐름 모호성 0
- **사용자 명시 원칙 영구 보존**: feedback_sprint-execution-style.md 메모리로 매 sprint 일관성 확보

### 6.2 What Needs Improvement (Problem)
- **bkit ENH-310 heredoc guard와의 friction**: commit message heredoc이 bkit 자체 보안에 차단됨 → heredoc-free workflow 또는 temp file 패턴 표준화 필요
- **AC-T12 Track B PR 미제출**: Wave 0 RCA로 Gemini CLI가 이미 'allow' 지원 입증 → Track B 우선순위 ↓. PR은 후속 작업으로 deferred (interactive mode 검증 후 사용자 결재)
- **AC-T13 mutation testing 별도 CI**: stryker-js는 본 sprint scope에서 적용 deferred. v2.1.0 CI matrix에 추가 필요

### 6.3 Try Next Sprint
- **Auto-checkpoint integration with before-tool**: 현재 checkpoint.js는 standalone module — before-tool hook에서 Edit/Write 시 자동 호출하는 통합은 S5 또는 별도 sprint에서
- **before-phase agent dispatch 실제 통합**: D15 stub을 실제 sprint-qa-flow MCP 호출로 upgrade (Gemini CLI hook async 지원 검증 후)
