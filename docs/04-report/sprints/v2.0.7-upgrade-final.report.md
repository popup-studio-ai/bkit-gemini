## 6. Lessons Learned & Retrospective

### 6.1 What Went Well (Keep)

- **Phase-Aware Validation**: Splitting testing into L1-L5 levels allowed for surgical verification of specific version flags.
- **Slim Context Strategy**: Reducing `GEMINI.md` to a "pointer" document significantly improved token efficiency while maintaining discoverability.
- **Dual-Version Simulation**: Using `GEMINI_CLI_VERSION` env var and `npx` allowed testing v0.42.0 features without upgrading the host CLI.

### 6.2 What Needs Improvement (Problem)

- **Manual LLM Verification**: Real-world hook behavior still requires `gemini -p` calls which are expensive and slow.
- **Baseline Stabilization**: Legacy test failures (83 cases) consume cognitive overhead during regression analysis.
