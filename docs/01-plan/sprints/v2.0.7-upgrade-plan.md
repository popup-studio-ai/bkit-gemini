## §7. Decision Records

### 7.1 본 sprint 결재 항목 (D-A1 / D-A2 / D-A3)

#### D-A1: M6 (topicUpdateNarration) 정당화 명문화 vs 해제

**Context**: 직전 sprint `1b452ef`에서 `.gemini/settings.json` 4 잠금 (gemma:false / autoMemory:false / memoryManager:false / topicUpdateNarration:false). 3 잠금은 GEMINI.md:37-42에 정당화 명시, M6 (topicUpdateNarration)은 *암묵 잠금*.

**Decision Options**:
1. **Option A (권장, ENH-2)**: GEMINI.md:41 1줄 → 3줄 확장 + bkit Phase-Aware Context Engineering의 *topic noise* 회피 사유 명시
2. **Option B**: 잠금 해제 (`.gemini/settings.json`에서 키 제거) — Gemini CLI default `true` 따름
3. **Option C**: 환경 변수 토글 (`BKIT_TOPIC_NARRATION=on/off`) — 사용자 선택 가능

**Rationale (Option A)**:
- 정당화 명문화는 *낮은 비용* (15분)
- Phase-Aware Context Engineering 컨셉이 *topic noise*를 의도적으로 억제하는 설계
- 사용자가 *원하면* 직접 `.gemini/settings.json`에서 해제 가능 (잠금이 *덮어쓰기*는 아니므로)

**결재자**: 사용자
**결재 시점**: design → do 전이
**상태**: ⏳ 대기

#### D-A2: ENH-4 (Auto Memory inbox 안내) 톤

**Context**: bkit이 `autoMemory: false` 잠금 유지하지만, 사용자가 *수동 활성화* 시 SessionStart에서 inbox 존재 안내. 안내 톤이 사용자 의사결정에 영향.

**Decision Options**:
1. **Option A (정보성)**: "Auto Memory inbox detected. bkit retains experimental.autoMemory:false by default."
2. **Option B (권유성)**: "Auto Memory inbox detected. Consider running `gemini memory inbox` to review."
3. **Option C (경고성)**: "WARNING: experimental.autoMemory is enabled outside bkit defaults."

**Rationale**:
- A = 사용자 의사결정 존중. 정보 제공만.
- B = bkit이 권유. 사용자가 *왜 권유받는지* 명확하지 않을 수 있음.
- C = 사용자가 *자신의 결정을 비난받는*다고 인지할 위험.

**결재자**: 사용자
**결재 시점**: design → do 전이
**상태**: ⏳ 대기

#### D-A3: PR merge

**Context**: 본 sprint 완료 후 PR 머지. commit `1b452ef` (호환성 base) + 본 sprint commits 단일 PR.

**Decision Options**:
