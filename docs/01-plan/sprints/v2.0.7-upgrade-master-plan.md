## §0. Executive Summary

### 0.1 Mission

**Gemini CLI v0.42.0 stable과 Google이 2025-12-17 발표한 "Conductor: context-driven development" 철학에 bkit이 *부합 + 차별화*하도록 v2.0.7을 고도화한다.** 직전 sprint(`v0.42.0-stable-migration`, commit `1b452ef`)가 호환성 base를 확립했으나 *4 설정 잠금*과 *Cx1~Cx14 13개 차단 결정*으로 인해 "차단/회피 방향"이라는 사용자 평가를 받았다. 본 sprint는 그 base 위에 *활용/고도화* commit을 누적해 단일 PR로 통합한다.

### 0.2 Anti-Mission (본 sprint scope 외)

- bkit agent dispatch 메커니즘 변경 (R-extra-1) — `v2.1.0-agent-dispatch-fix` sprint (별도)
- `GEMINI_SYSTEM_MD` firmware 패턴 도입 (scope 큼, ~4h+) — 별도 sprint
- `/export-session` / `--session-file` 활용 — v0.43.0 stable 출시 대기
- Auto Memory `autoMemory: true` 본격 활성화 (옵트인 모드) — `v2.1.0-context-optimization` sprint
- 사전 부채 83건 회복 — `bkit-baseline-stabilization` sprint
- 1H prompt caching 자동화 — v2.1.0
- bkit `i18n` / Cross-LLM portability 재설계 — 별도 cycle

### 0.3 4-Perspective Value

| 관점 | 가치 |
|---|---|

## §1. Context Anchor

### 1.1 WHY (왜 본 sprint를 진행하는가)

직전 sprint `v0.42.0-stable-migration` (commit `1b452ef`) PR 검토 중 사용자(2026-05-14) 평가:

> "gemini도 버전업 되면서 신규 기능등 더 고도화 하고 있는데 나는 거의 차단하는 방향으로 하고 있네? v0.42.0로 개선된 gemini에 부합하도록 bkit도 고도화하고 싶은건데... v2.0.7 방향이 애초에 잘못 된 것 같아. gemini cli의 공식문서, 기술 블로그, 깃허브(이슈 포함), gemini cli가 추구하는 Context Engineering 방향까지 모두 심층 조사하고 이 코드베이스도 심층 조사하여 bkit extensions 고도화로 사용자 경험이 개선되도록 하고 싶어."

5축 심층 조사(~2h) 결과:

1. **D축 핵심 발견** — Google "Conductor: context-driven development" (2025-12-17 developers.googleblog.com) = bkit PDCA + Phase-Aware Context Engineering의 *우연이 아닌 수렴*. bkit이 1년 먼저 도달. 그러나 *bkit이 이 동형성을 명문화하지 않음* = 가치 명제 신뢰도 손실.

2. **D축 매트릭스 21항목 결과** — 적합 8 / 부분 적합 5 / 충돌 4 / 중립 4. 충돌 4건 중 **3건은 정당** (M3 autoMemory, M4 memoryManager, M7 gemma — GEMINI.md:37-42에 정당화 명시), **1건은 정당화 부재** (M6 topicUpdateNarration — *암묵 잠금*) = 안티패턴 A1.

3. **E축 6 ENH 도출** — A/B/C/D 권고를 종합해 본 sprint 채택 가능한 6건(P0×3 + P1×3)으로 압축. ~4h, 위험 LOW.

4. **C축 28 PRs 직접 영향** — Critical 9건 / High 13건 / Medium 6건. 그중 본 sprint 안에 즉시 흡수해야 할 것은 capability flag 차원에서 다룬다.

5. **B축 외부 모범 사례** — "vibe coding is passé → agentic engineering" (Karpathy 2025-09) 등 트렌드 시그널을 carry forward.

본 sprint는 위 발견을 **단일 PR 단일 sprint**로 통합해 v2.0.7 출시한다.
