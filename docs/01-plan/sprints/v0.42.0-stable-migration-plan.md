## §1. Sprint Scope Recap

### 1.1 Sprint vs PDCA 결정 근거

본 마이그레이션은 **PDCA per-feature (9-Phase × 5 cycle)** 와 **Sprint container (8-Phase × 1)** 양자 선택지가 있었다. Sprint 채택 결정 근거:

| 비교 항목 | PDCA × 5 cycle | **Sprint × 1 (본 채택)** |
|---|---|---|
| PR 단위 | 5개 (cycle별) | **1개** |
| 사용자 결재 | ~20건 (cycle별 D1~D5) | **4건** (D1/D3/D4/D5) |
| 총 시간 | ~25h (5 × ~5h × redundancy) | **≤ 5h** |
| token | ~250K (5 × ~50K) | **< 50K** |
| 회귀 검증 | 5회 (21 agent smoke × 5) | **1회** |
| 미머지 plan 부담 | 누적 4건 + 본 cycle = 5건 | **0건** (단일 PR) |
| 결재 분산 | 5 cycle 분산 | **단일 sprint 집중** |
| 4 Auto-Pause | 없음 | **있음** (QUALITY_GATE_FAIL / ITERATION_EXHAUSTED / BUDGET_EXCEEDED / PHASE_TIMEOUT) |

**결론**: Sprint 채택. master plan §0.3 4-Perspective Value의 "Solution"이 본 비교의 합리화 기반.
