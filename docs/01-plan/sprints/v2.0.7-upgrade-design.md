```markdown
**Key behavior locks** (`.gemini/settings.json` — pinned by this extension):
- `experimental.gemma: false` — prevents Gemini CLI v0.42.0's Gemma 4 default-on regression (Cx13 / [PR #26307](...)). Users wanting Gemma must opt in explicitly.
- `experimental.autoMemory: false` — Auto Memory inbox flow (Cx2, introduced in v0.41.0) is opt-in; bkit defers adoption to the v2.1.0 cycle.
- `experimental.memoryManager: false` — memoryManager (v0.41.x) is opt-in for the same reason.
- `general.topicUpdateNarration: false` — topic-narration noise suppression.
- `experimental.enableAgents: true` — bkit agent catalog visibility (existing setting, unchanged).
```

**문제**: `general.topicUpdateNarration: false` 항목이 *"topic-narration noise suppression"* 1줄로 끝남 — 다른 3 잠금에 비해 정당화 빈약 (D.5 §M6 / E.3.2 / 안티패턴 A1).

#### 2.1.3 변경 후 (3줄 확장, D-A1 Option A 선택 시)

```markdown
- `general.topicUpdateNarration: false` — Suppresses Gemini CLI's automatic topic-change narration messages.
  - **Rationale 1**: bkit's Phase-Aware Context Engineering tracks phase transitions explicitly via `PHASE_CONTEXT_MAP` (hooks/scripts/session-start.js:260). Native narration would duplicate this signal as user-facing noise.
  - **Rationale 2**: PHASE_CONTEXT_MAP's deterministic state transitions are orthogonal to LLM-inferred topic shifts. Mixing the two could cause confusion when a phase transition (PDCA-driven) coincides with a topic shift (conversation-driven).
  - **Rationale 3**: Users who prefer the upstream narration can override by removing this key from `.gemini/settings.json` or setting `general.topicUpdateNarration: true` (bkit does not enforce; this is a default).
  - **Reference**: PR #25586 (v0.40.0, promote narration to default-on)
```

#### 2.1.4 결재 처리

- **D-A1 Option A**: 위 3줄 확장 (권장)
- **D-A1 Option B**: 잠금 해제 (`.gemini/settings.json`에서 `general.topicUpdateNarration` 키 제거)
- **D-A1 Option C**: 환경 변수 토글 (`BKIT_TOPIC_NARRATION` 신설) — 추가 1h 작업

본 design은 Option A 가정으로 작성. D-A1 결재 결과에 따라 변경.
