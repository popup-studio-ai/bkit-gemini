# Sprint v2.0.7-context-engineering-integration — Design

> **Sprint ID**: S5 | **Date**: 2026-05-15

## 1. 4-Wave Architecture

```
Wave 1 — Compaction
  hooks/scripts/pre-compress.js (+50L)
    ↓ uses
  lib/core/compactor.js (NEW ~210L)
    ├─ shouldCompact(state) — turn / token threshold
    ├─ defaultExtractDecisions(payload) — regex-based heuristic
    ├─ saveSnapshot(projectDir, snapshot) — .bkit/snapshots/compact-<ts>.json
    ├─ compactSession(state, projectDir) — E2E entry
    └─ setExtractor(fn) — injection point for future LLM extractor

Wave 2 — Tool-result clearing
  lib/core/tool-result-summarizer.js (NEW ~150L)
    ├─ shouldSummarize({toolName, result}) — size + tool gate
    ├─ summarize(args) — HEAD + TAIL + refs structured marker
    ├─ extractRefs(text) — file:line discovery
    └─ applyClearing(args) — high-level pass-through or summarize

Wave 3 — Memory tool extraction.patch
  lib/core/agent-memory.js (+120L)
    ├─ exportAsExtractionPatch() — sessions + patterns → bkit-agent-memory/1.0 schema
    └─ importFromExtractionPatch(patch) — merge sessions (dedupe) + patterns (union)

Wave 4 — Sub-agent isolation
  lib/gemini/agent-dispatch.js (+60L)
    ├─ wrapAsIsolated(dispatchCall) — isolated:true + isolationPolicy:context-fork
    └─ detectAndBuildIsolated(prompt) — E2E entry (detect + build + wrap)
```

## 2. Environment Toggles

| Env Var | Default | Behavior |
|---------|---------|----------|
| `BKIT_COMPACTION` | true | `false`/`0`/`off` → compactor disabled |
| `BKIT_TOOL_RESULT_CLEARING` | true | `false`/`0`/`off` → summarizer disabled |

(W3 / W4는 항상 활성, opt-out 불필요 — round-trip 안전 + 메타데이터만 추가)

## 4. Cross-Wave Integration

- **pre-compress.js** (Wave 1) → PDCA status에서 history/notes 추출 → compactor 호출 → `.bkit/snapshots/<ts>.json.compact.json` 작성
- **after-tool.js** (Wave 2) → 별도 wave 1과 분리 hook이므로 통합은 incremental — 현재 framework 입증만
- **agent-memory.js** (Wave 3) → AgentMemoryManager의 export/import는 caller 책임으로 save() 호출. 21 agent 기존 file 무손상 (MX-09 검증).
- **agent-dispatch.js** (Wave 4) → detectAndBuildIsolated은 S3의 detectDispatch/buildDispatchCall 위 wrapper. before-model.js 통합은 incremental.
