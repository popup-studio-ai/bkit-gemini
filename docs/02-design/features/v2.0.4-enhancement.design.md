## 1. Design Overview

### 1.1 배경 및 목표

v2.0.3에서 확인된 핵심 문제:

| 문제 | 현황 | 영향 |
|------|------|------|
| Dead Code 과다 | lib/team(12), lib/task(10), lib/intent(5), skill-orchestrator(754L), context-hierarchy(209L) | 유지보수 비용 증가, import confusion |
| Empty Skill Stubs | 8개 스킬 디렉토리가 비어있음 (audit, control, deploy 등) | 사용자 혼란, coverage gap |
| MCP Server 미등록 | spawn-agent-server.js(907L) 존재하나 settings.json에 미등록 | 핵심 기능 비활성 |
| Hook 한계 | deny only, no parallel, no recursion | 복잡한 workflow 불가 |
| Orchestration 부재 | PM/QA/Iterate pipeline이 skill.md 텍스트로만 존재 | 실행 자동화 불가 |

**v2.0.4 목표**: MCP Server를 orchestration engine으로 확립하고, dead code를 제거하며,
empty skill stub을 MCP tool 기반으로 채운다.

### 1.4 Architecture Options 비교

#### Option A: Single MCP Server 확장 (spawn-agent-server.js 확장)

기존 `mcp/spawn-agent-server.js`에 6개 tool을 직접 추가하는 방식.

**장점:**
- 변경 범위 최소 (1 파일 수정)
- 기존 JSON-RPC infra 재사용
- 배포/테스트 단순

**단점:**
- 907L → 1,500L+ 으로 단일 파일 비대화
- spawn_agent 관심사와 workflow tool 관심사 혼합
- 향후 tool 추가 시 파일 관리 어려움
- server name이 `bkit-agents`로 제한적

#### Option B: Multiple MCP Servers (3개 분리)

```json
{
  "mcpServers": {
    "bkit-agents":   { "args": ["mcp/spawn-agent-server.js"] },
    "bkit-workflow": { "args": ["mcp/workflow-server.js"] },
    "bkit-analysis": { "args": ["mcp/analysis-server.js"] }
  }
}
```

**장점:**
- 관심사 완전 분리
- 독립 배포/업데이트 가능
- 서버별 독립 재시작

**단점:**
- 3개 Node.js process → memory overhead (3x ~50MB)
- Gemini CLI의 MCP server 수 제한 가능성 (undocumented)
- 서버 간 상태 공유 필요 (file-based IPC)
- 설정 복잡도 증가

#### Option C: Single Unified bkit MCP Server (교체) -- Recommended

기존 `spawn-agent-server.js`를 `bkit-server.js`로 교체 (rename + 확장).
내부 로직은 `mcp/tools/*.js` 모듈로 분리.

```
mcp/
  bkit-server.js          (~800-1000L) - MCP protocol + tool router
  tools/
    pm-pipeline.js        (~200L)  - PM 분석 pipeline
    qa-runner.js           (~250L)  - QA test 실행
    gap-analyzer.js        (~200L)  - Design vs Implementation 비교
    audit-store.js         (~100L)  - JSONL audit read/write
    checkpoint-manager.js  (~100L)  - Checkpoint CRUD
```

**장점:**
- 단일 process, 단일 등록 → 설정 단순
- 내부 모듈 분리로 관심사 분리 달성
- server name `bkit`으로 통일 → tool 이름 자연스러움
- 기존 spawn_agent 기능 100% 유지
- file-level modularity로 테스트 용이

**단점:**
- spawn-agent-server.js 삭제 필요 (migration 필요)
- 초기 작업량이 Option A보다 많음

### 1.5 Options 비교 테이블

| 기준 | Option A (확장) | Option B (분리) | Option C (통합) |
|------|----------------|----------------|----------------|
| Process 수 | 1 | 3 | 1 |
| Memory 사용 | ~50MB | ~150MB | ~55MB |
| 파일 변경 수 | 1 | 4+ | 7 |
| 관심사 분리 | X | O | O (모듈) |
| 설정 복잡도 | 낮음 | 높음 | 낮음 |
| 확장성 | 낮음 | 높음 | 높음 |
| Migration 난이도 | 쉬움 | 어려움 | 보통 |
| Gemini CLI 호환성 | 안전 | 위험 (multi-server) | 안전 |
| **권장** | | | **O** |

**결정: Option C 채택**

이유: 단일 process로 Gemini CLI 호환성을 보장하면서,
`mcp/tools/` 모듈화로 관심사 분리와 테스트 용이성을 확보.

## 2. MCP Server Design -- bkit-server.js

### 2.1 Server Registration

**파일**: `.gemini/settings.json`

```json
{
  "experimental": {
    "enableAgents": true
  },
  "mcpServers": {
    "bkit": {
      "command": "node",
      "args": ["${extensionPath}/mcp/bkit-server.js"],
      "cwd": "${workspaceFolder}"
    }
  }
}
```

> **Gemini CLI 제약**: `${extensionPath}`는 Gemini CLI가 extension 디렉토리로 resolve.
> `${workspaceFolder}`는 현재 워크스페이스 루트. Gemini CLI v0.36.0 이상 필수.

### 2.2 Server Core Structure

`mcp/bkit-server.js`는 기존 `SpawnAgentServer` class를 `BkitServer`로 rename하고,
12개 tool (기존 6 + 신규 6)을 router로 처리한다.

const fs = require('fs');

async function run({ feature, projectDir, phases, extensionPath }) {
  const context = {};
  const results = { feature, phases: [], errors: [] };

  // Step 1: Project Context 수집
  context.packageJson = safeReadJson(path.join(projectDir, 'package.json'));
  context.readme = safeReadFile(path.join(projectDir, 'README.md'));
  context.gitLog = await execShell('git log --oneline -20', projectDir);
  context.fileStructure = await execShell('find . -type f -name "*.js" -o -name "*.ts" | head -50', projectDir);

  // Step 2: Phase 별 실행
  if (phases.includes('discovery')) {
    const discovery = runDiscovery(context, feature);
    results.phases.push({ name: 'discovery', ...discovery });
    context.discovery = discovery;
  }

  if (phases.includes('strategy')) {
    const strategy = runStrategy(context, feature);
    results.phases.push({ name: 'strategy', ...strategy });
    context.strategy = strategy;
  }

  if (phases.includes('research')) {
    const research = runResearch(context, feature);
    results.phases.push({ name: 'research', ...research });
    context.research = research;

  return results;
}

#### Implementation Pseudo-code

```javascript
// mcp/tools/qa-runner.js
const { execSync, spawnSync } = require('child_process');
const fs = require('fs');
const path = require('path');
const { glob } = require('./utils/glob-helper');

async function run({ feature, projectDir, levels = [1, 2], testPattern }) {
  const results = {
    feature,
    timestamp: new Date().toISOString(),
    framework: null,
    levels: [],
    summary: { total: 0, passed: 0, failed: 0, skipped: 0 },
    passRate: 0,
    coverage: null
  };

  // Step 1: Test framework 자동 감지
  results.framework = detectFramework(projectDir);

  // Step 2: Level 별 테스트 실행
  for (const level of levels.sort()) {
    const levelResult = await runLevel(level, {
      feature, projectDir, framework: results.framework, testPattern
    });
    results.levels.push(levelResult);
    results.summary.total += levelResult.total;
    results.summary.passed += levelResult.passed;
    results.summary.failed += levelResult.failed;
    results.summary.skipped += levelResult.skipped;
  }

  // Step 3: Pass rate 계산
  if (results.summary.total > 0) {
    results.passRate = Math.round(
      (results.summary.passed / results.summary.total) * 100
    );
  }

  // Step 4: Coverage 수집 (가능한 경우)
  if (results.framework && levels.includes(1)) {
    results.coverage = collectCoverage(projectDir, results.framework);
  }

  // Step 5: QA report 작성
  const reportPath = path.join(projectDir, 'docs', '03-test', `${feature}.qa-report.md`);
  fs.mkdirSync(path.dirname(reportPath), { recursive: true });
  fs.writeFileSync(reportPath, generateReport(results), 'utf-8');
  results.reportPath = reportPath;

  return results;
}

function detectFramework(projectDir) {
  const pkg = safeReadJson(path.join(projectDir, 'package.json'));
  if (!pkg) return null;

  const deps = { ...pkg.dependencies, ...pkg.devDependencies };

  if (deps['vitest'])                return 'vitest';
  if (deps['jest'])                  return 'jest';
  if (deps['@playwright/test'])      return 'playwright';
  if (deps['mocha'])                 return 'mocha';

  // Node.js built-in test runner 감지
  const scripts = pkg.scripts || {};
  if (Object.values(scripts).some(s => s.includes('node --test'))) {
    return 'node-test';
  }

  return null;
}

async function runLevel(level, { feature, projectDir, framework, testPattern }) {
  const levelNames = { 1: 'Unit', 2: 'API', 3: 'E2E', 4: 'UX', 5: 'Data' };
  const result = {
    level,
    name: levelNames[level],
    total: 0, passed: 0, failed: 0, skipped: 0,
    errors: [],
    duration: 0
  };

  const startTime = Date.now();

  try {
    switch (level) {
      case 1: // Unit Tests
        await runUnitTests(result, { projectDir, framework, testPattern, feature });
        break;

      case 2: // API Tests
        await runApiTests(result, { projectDir, feature });
        break;

      case 3: // E2E Tests
        await runE2ETests(result, { projectDir, feature });
        break;

      case 4: // UX Tests (lighthouse, accessibility)
        await runUxTests(result, { projectDir, feature });
        break;

      case 5: // Data Integrity Tests
        await runDataTests(result, { projectDir, feature });
        break;
    }
  } catch (err) {
    result.errors.push(err.message);
  }

  result.duration = Date.now() - startTime;
  return result;
}

async function runUnitTests(result, { projectDir, framework, testPattern, feature }) {
  const pattern = testPattern || `**/*${feature}*.test.{js,ts,jsx,tsx}`;
  const testFiles = glob(pattern, projectDir);

  if (testFiles.length === 0) {
    result.skipped = 1;
    result.total = 1;
    return;
  }

  const commands = {
    'jest':      `npx jest --json --passWithNoTests ${testFiles.join(' ')}`,
    'vitest':    `npx vitest run --reporter=json ${testFiles.join(' ')}`,
    'mocha':     `npx mocha --reporter=json ${testFiles.join(' ')}`,
    'node-test': `node --test --experimental-test-snapshots ${testFiles.join(' ')}`,
    null:        `node --test ${testFiles.join(' ')}`
  };

  const cmd = commands[framework] || commands[null];
  const output = safeExec(cmd, projectDir);
  parseTestOutput(output, framework, result);
}

async function runApiTests(result, { projectDir, feature }) {
  // HTTP endpoint 감지 (Express/Fastify/Koa 라우트 스캔)
  const routeFiles = glob(`**/*route*.{js,ts}`, projectDir);
  if (routeFiles.length === 0) {

// mcp/tools/gap-analyzer.js
const fs = require('fs');
const path = require('path');

function analyze({ feature, projectDir, designPath, extensionPath }) {
  const result = {
    feature,
    matchRate: 0,
    gaps: [],
    structural: { matched: 0, total: 0 },
    functional: { matched: 0, total: 0 },
    apiContract: { matched: 0, total: 0 }
  };

  // Step 1: Design document 파싱
  const designContent = safeReadFile(designPath);
  if (!designContent) {
    result.gaps.push({
      type: 'missing',
      severity: 'critical',
      description: `Design document not found: ${designPath}`,
      file: designPath,
      line: 0
    });
    return result;
  }

  const designSpec = parseDesignDocument(designContent);

  // Step 2: Structural Match - 파일 존재 여부
  for (const expectedFile of designSpec.files) {
    result.structural.total++;
    const filePath = path.join(projectDir, expectedFile.path);
    if (fs.existsSync(filePath)) {
      result.structural.matched++;
    } else {
      result.gaps.push({
        type: 'structural',
        severity: 'high',
        description: `Expected file missing: ${expectedFile.path}`,
        file: expectedFile.path,
        line: 0,
        expected: expectedFile.purpose
      });
    }
  }

  // Step 3: Functional Match - 함수/클래스 존재 여부
  for (const expectedFunc of designSpec.functions) {
    result.functional.total++;
    const filePath = path.join(projectDir, expectedFunc.file);
    if (!fs.existsSync(filePath)) continue;

    const content = fs.readFileSync(filePath, 'utf-8');
    const lines = content.split('\n');
    const found = lines.findIndex(line =>
      line.includes(expectedFunc.name) &&
      (line.includes('function') || line.includes('class') ||
       line.includes('=>') || line.includes('async'))
    );

    if (found >= 0) {
      result.functional.matched++;
    } else {
      result.gaps.push({
        type: 'functional',
        severity: 'high',
        description: `Expected function/class not found: ${expectedFunc.name}`,
        file: expectedFunc.file,
        line: 0,
        expected: expectedFunc.signature
      });
    }
  }

  // Step 4: API Contract Match - endpoint/schema 검증
  for (const contract of designSpec.apiContracts) {
    result.apiContract.total++;
    // route 파일에서 endpoint 패턴 검색
    const found = searchPattern(projectDir, contract.pattern);
    if (found) {
      result.apiContract.matched++;
    } else {
      result.gaps.push({
        type: 'api_contract',
        severity: 'medium',
        description: `API contract missing: ${contract.method} ${contract.path}`,
        file: contract.expectedFile || 'unknown',
        line: 0,
        expected: contract
      });
    }
  }

  // Step 5: Match Rate 계산
  const totalChecks = result.structural.total + result.functional.total + result.apiContract.total;
  const totalMatched = result.structural.matched + result.functional.matched + result.apiContract.matched;
  result.matchRate = totalChecks > 0 ? Math.round((totalMatched / totalChecks) * 100) : 0;

  return result;
}

function parseDesignDocument(content) {
  // Design document에서 구조적 정보 추출
  // "### 3.1 New Files" 섹션 → 파일 목록
  // "### Function Signatures" → 함수 목록
  // "### API Endpoints" → API contract 목록
  const spec = { files: [], functions: [], apiContracts: [] };

  // 파일 목록 추출 (- path/to/file.js 패턴)
  const filePattern = /^[-*]\s+`?([a-zA-Z0-9_\-/.]+\.[a-zA-Z]+)`?\s*[-—(]?\s*(.*)/gm;
  let match;
  while ((match = filePattern.exec(content)) !== null) {
    if (match[1].match(/\.(js|ts|jsx|tsx|py|go|rs)$/)) {
      spec.files.push({ path: match[1], purpose: match[2] || '' });
    }
  }

  // 함수 시그니처 추출
  const funcPattern = /(?:function|async function|class)\s+(\w+)/g;
  while ((match = funcPattern.exec(content)) !== null) {
    spec.functions.push({ name: match[1], file: '', signature: match[0] });
  }

  return spec;
}

function searchPattern(projectDir, pattern) {
  // grep -r 대용: 파일 시스템에서 패턴 검색
  try {
    const { execSync } = require('child_process');
    const result = execSync(
      `grep -r "${pattern}" --include="*.js" --include="*.ts" -l`,
      { cwd: projectDir, encoding: 'utf-8', timeout: 10000 }
    );
    return result.trim().length > 0;
  } catch { return false; }
}

const fs = require('fs');
const path = require('path');

function query({ projectDir, eventType, keyword, since, limit = 50 }) {
  const auditDir = path.join(projectDir, '.gemini', 'audit');
  if (!fs.existsSync(auditDir)) {
    return { entries: [], total: 0, message: 'No audit directory found' };
  }

  // JSONL 파일 목록 (최신 순)
  const files = fs.readdirSync(auditDir)
    .filter(f => f.endsWith('.jsonl'))
    .sort()
    .reverse();

  const entries = [];
  const sinceDate = since ? new Date(since) : null;

  for (const file of files) {
    if (entries.length >= limit) break;

    const lines = fs.readFileSync(path.join(auditDir, file), 'utf-8')
      .split('\n')
      .filter(l => l.trim());

    for (const line of lines.reverse()) {
      if (entries.length >= limit) break;

      try {
        const entry = JSON.parse(line);

        // Filter: eventType
        if (eventType && entry.type !== eventType) continue;

        // Filter: since
        if (sinceDate && new Date(entry.timestamp) < sinceDate) continue;

        // Filter: keyword
        if (keyword && !JSON.stringify(entry).toLowerCase().includes(keyword.toLowerCase())) continue;

        entries.push(entry);
      } catch { /* skip malformed lines */ }
    }
  }

  return { entries, total: entries.length };
}

function append({ projectDir, entry }) {
  const auditDir = path.join(projectDir, '.gemini', 'audit');

  const filePath = path.join(auditDir, `${today}.jsonl`);

  const record = {
    timestamp: new Date().toISOString(),
    ...entry
  };

  fs.appendFileSync(filePath, JSON.stringify(record) + '\n');

const fs = require('fs');
const path = require('path');
const { execSync } = require('child_process');

function execute({ action, feature, projectDir, checkpointId, label }) {
  const checkpointDir = path.join(projectDir, '.bkit', 'checkpoints');
  fs.mkdirSync(checkpointDir, { recursive: true });

  switch (action) {
    case 'create': {
      const id = `chk-${Date.now()}`;
      const gitHash = safeExec('git rev-parse HEAD', projectDir).trim();
      const gitDiff = safeExec('git diff --stat', projectDir);

      const checkpoint = {
        id,
        feature: feature || 'unknown',
        label: label || `Checkpoint at ${new Date().toISOString()}`,
        gitHash,
        gitDiff,
        createdAt: new Date().toISOString(),
        files: getTrackedFiles(projectDir)
      };

      fs.writeFileSync(
        path.join(checkpointDir, `${id}.json`),
        JSON.stringify(checkpoint, null, 2)
      );
      return { success: true, checkpoint };
    }

    case 'list': {
      const files = fs.readdirSync(checkpointDir).filter(f => f.endsWith('.json'));
      const checkpoints = files.map(f => {
        const data = JSON.parse(fs.readFileSync(path.join(checkpointDir, f), 'utf-8'));
        return { id: data.id, feature: data.feature, label: data.label, createdAt: data.createdAt };
      }).sort((a, b) => b.createdAt.localeCompare(a.createdAt));
      return { success: true, checkpoints, total: checkpoints.length };
    }

    case 'restore': {
      if (!checkpointId) return { success: false, error: 'checkpointId required' };
      const chkPath = path.join(checkpointDir, `${checkpointId}.json`);
      if (!fs.existsSync(chkPath)) return { success: false, error: `Checkpoint not found: ${checkpointId}` };

      const checkpoint = JSON.parse(fs.readFileSync(chkPath, 'utf-8'));
      // git stash → git checkout → confirm
      safeExec('git stash', projectDir);
      safeExec(`git checkout ${checkpoint.gitHash}`, projectDir);
      return { success: true, restoredTo: checkpoint.gitHash, label: checkpoint.label };
    }

    case 'delete': {
      if (!checkpointId) return { success: false, error: 'checkpointId required' };
      const chkPath = path.join(checkpointDir, `${checkpointId}.json`);
      if (!fs.existsSync(chkPath)) return { success: false, error: `Checkpoint not found: ${checkpointId}` };
      fs.unlinkSync(chkPath);
      return { success: true, deleted: checkpointId };
    }

    default:
      return { success: false, error: `Unknown action: ${action}` };
  }
}

function getTrackedFiles(projectDir) {
  const output = safeExec('git ls-files', projectDir);
  return output.split('\n').filter(f => f.trim()).length;
}

function safeExec(cmd, cwd) {
  try { return execSync(cmd, { cwd, encoding: 'utf-8', timeout: 10000 }); }
  catch (e) { return e.message; }
}

module.exports = { execute };
```

## 8. MCP Internal Module Design

### 8.1 mcp/tools/pm-pipeline.js

```
Exports:
  run({ feature, projectDir, phases, extensionPath }) → Promise<PMResult>

Dependencies:
  - fs, path (Node.js built-in)
  - child_process.execSync (for git log, find)

Data Flow:
  package.json + README + git log
    → runDiscovery() → discovery context
    → runStrategy(context) → strategy context
    → runResearch(context) → research context
    → synthesizePRD(all context) → PRD markdown
    → write to docs/00-pm/{feature}.prd.md
    → return PMResult { status, phases[], prdPath }

Internal Functions:
  - safeReadJson(filePath: string): object | null
  - safeReadFile(filePath: string): string | null
  - execShell(cmd: string, cwd: string): Promise<string>
  - detectTechStack(packageJson: object): string[]
  - extractFeatures(readme: string): string[]
  - parseGitLog(log: string): object[]
  - runDiscovery(context, feature): DiscoveryResult
  - runStrategy(context, feature): StrategyResult
  - runResearch(context, feature): ResearchResult
  - synthesizePRD(context, feature): string (markdown)
```

### 8.2 mcp/tools/qa-runner.js

```
Exports:
  run({ feature, projectDir, levels, testPattern }) → Promise<QAResult>

Dependencies:
  - fs, path (Node.js built-in)
  - child_process.execSync, spawnSync

Data Flow:
  package.json → detectFramework()
    → L1: runUnitTests() → jest/vitest/mocha JSON output
    → L2: runApiTests() → curl health check + endpoint smoke
    → L3: runE2ETests() → playwright JSON output
    → L4: runUxTests() → lighthouse/axe report
    → L5: runDataTests() → schema validation
    → collectCoverage() → coverage stats
    → generateReport() → markdown
    → write to docs/03-test/{feature}.qa-report.md
    → return QAResult { passRate, summary, levels[], reportPath }

Internal Functions:
  - detectFramework(projectDir): string | null
  - runLevel(level, options): Promise<LevelResult>
  - runUnitTests(result, options): void (mutates result)
  - runApiTests(result, options): void
  - runE2ETests(result, options): void
  - runUxTests(result, options): void
  - runDataTests(result, options): void
  - collectCoverage(projectDir, framework): CoverageResult | null
  - generateReport(results): string (markdown)
  - parseTestOutput(output, framework, result): void
  - parsePlaywrightOutput(output, result): void
```

### 8.3 mcp/tools/gap-analyzer.js

```
Exports:
  analyze({ feature, projectDir, designPath, extensionPath }) → GapResult

Dependencies:
  - fs, path (Node.js built-in)
  - child_process.execSync (for grep)

Data Flow:
  design.md → parseDesignDocument()
    → structural check: file existence
    → functional check: function/class presence in source
    → API contract check: endpoint pattern search
    → calculate matchRate
    → return GapResult { matchRate, gaps[], structural, functional, apiContract }

Internal Functions:
  - parseDesignDocument(content: string): DesignSpec
  - searchPattern(projectDir: string, pattern: string): boolean
  - safeReadFile(filePath: string): string | null
```

### 8.4 mcp/tools/audit-store.js

```
Exports:
  query({ projectDir, eventType, keyword, since, limit }) → QueryResult
  append({ projectDir, entry }) → AuditRecord

Dependencies:
  - fs, path (Node.js built-in)

Data Flow:
  query:
    .gemini/audit/*.jsonl → read + filter → return entries[]
  append:
    entry → JSON.stringify → appendFileSync → .gemini/audit/{date}.jsonl
```

### 8.5 mcp/tools/checkpoint-manager.js

```
Exports:
  execute({ action, feature, projectDir, checkpointId, label }) → CheckpointResult

Dependencies:
  - fs, path (Node.js built-in)
  - child_process.execSync (for git rev-parse, git diff, git stash, git checkout)

Data Flow:
  create:
    git rev-parse HEAD + git diff --stat → checkpoint JSON → .bkit/checkpoints/{id}.json
  list:
    .bkit/checkpoints/*.json → parse + sort → return checkpoints[]
  restore:
    checkpoint.gitHash → git stash → git checkout → return result
  delete:
    .bkit/checkpoints/{id}.json → unlinkSync → return result
```

## Appendix B: Gemini CLI Constraints Reference

v2.0.4 설계 전반에 걸쳐 고려된 Gemini CLI 제약사항:

| 제약 | 영향 | 대응 |
|------|------|------|
| Hook은 deny/allow만 반환 | Orchestration 불가 | MCP server에서 처리 |
| Hook은 parallel 실행 불가 | 동시 agent 불가 | MCP server sequential 처리 |
| Hook은 recursion 불가 | Self-referencing workflow 불가 | MCP iterationState로 대체 |
| MCP server는 별도 process | 메모리 격리 | File-based state (.gemini/, .bkit/) |
| Agent는 single-turn per spawn | 대화형 워크플로우 불가 | Task description에 전체 context 포함 |
| `--approval-mode` flag | 안전 tier 구현 | SAFETY_TIERS enum으로 매핑 |
| `--allowed-tools` flag (v0.36.0+) | READONLY agent tool 제한 | Multi-Registry 활용 |
| `${extensionPath}` 변수 | 동적 경로 | settings.json에서 사용 |
| JSONL transport | 메시지 크기 제한 | 대용량 결과는 파일 기록 후 경로 반환 |
