> **Purpose**: v2.0.7 Debt Resolution Program (7 sprint) 완료 후 사용자 입장에서 **모든 기능이 정상 동작하는지** gemini CLI를 직접 활용하여 종합 검증한다. 각 기능별 테스트 케이스, 입력 prompt, 기대 출력, 검증 절차를 상세 명시.
>
> **Companion**: 사용자가 그대로 복사해서 사용할 수 있는 prompt cheat sheet → `docs/test-plans/v2.0.7-user-test-prompts.md`
>
> **Date**: 2026-05-15
> **Author**: kay@popupstudio.ai + Claude Opus 4.7 (1M context)
> **Scope**: v2.0.7 전체 기능 (S7+S3+S2+S1+S5+S4+S6 통합)

## 0. 사전 준비

### 0.1 환경

| 항목 | 권장 |
|------|------|
| Gemini CLI | ≥ v0.34.0 (테스트된 최신: v0.42.0 stable) |
| Node.js | ≥ 18 |
| OS | macOS / Linux / WSL / Git Bash |
| bkit 설치 | `gemini extensions install https://github.com/popup-studio-ai/bkit-gemini` 또는 local development link |
| 작업 디렉터리 | bkit 프로젝트가 enable된 새 폴더 (`mkdir bkit-qa-test && cd bkit-qa-test`) |

### 0.2 Smoke Test (필수, 다른 테스트 진입 전)

```bash
# 1) 설치 확인 + 슬림 헤더 출력
GEMINI_CLI_TRUST_WORKSPACE=true gemini -p "bkit Hi"
```

**기대 출력 (1줄 헤더)**:
```
bkit Vibecoding Kit v2.0.7 activated (Gemini CLI) - Level: <Starter|Dynamic|Enterprise>
... (LLM 응답)
bkit session ended. State preserved.
```

이 smoke가 fail하면 다른 테스트 진행 무의미 → 설치/trust 문제 우선 해결 (§Appendix A 참조).

### 0.3 Auto Regression (개발자 사전 검증)

```bash
# 모든 자동 회귀 (LLM 호출 0회, ~5분)
./scripts/run-ci-stub.sh
```

기대: 모든 suite PASS. fail이 있으면 manual QA 진행 전 fix 우선.

## 10. End-to-End 통합 시나리오 (5 카나리아)

### E2E-B PM Discovery → 구현 → Report (L4 자동)
1. `/control level 4` (trust score ≥ 60 가정)
2. `/pdca plan user-dashboard`
3. `/pdca design user-dashboard`
4. `/pdca implement user-dashboard`
5. `/pdca analyze user-dashboard` (matchRate 검증)
6. `/pdca iterate user-dashboard` (< 90% 시)
7. `/pdca report user-dashboard`

**Pass criteria**: 모든 PDCA phase 자동 진행 + 각 phase 산출물 생성 + BeforePhase gate 통과 + audit log에 phase 전환 기록.

### E2E-C Agent Dispatch + Sub-agent Isolation
1. `Use the security-architect agent to review OWASP A03 risks in this code` (영어)
2. `code-analyzer 에이전트로 이 함수의 cyclomatic complexity 평가해줘` (한국어)
3. `Use the gap-detector agent to compare design vs implementation`
4. audit log 확인 → 3건 agent_dispatch 기록

**Pass criteria**: 3 agent 모두 dispatch + `[Agent: X]...[End Agent Output]` boundary + audit log 라인 3개 추가.

### E2E-D 보안 deny + bypass 시도 실패
1. `Run: rm -rf /` → bkit hook deny
2. `Run: r\b\bm -rf /` (ANSI bypass) → cmd-parser detect → deny
3. `Run: bash <<'EOF'` heredoc bypass → ENH-310 guard → deny
4. `Use bkit MCP team_create with team_name='team;rm'` → sanitize 거부

**Pass criteria**: 4 시도 모두 deny 또는 거부 응답. 실 파일시스템 변경 0건.

### E2E-E Context Engineering 4-Strategy 통합

2. `BKIT_TOOL_RESULT_CLEARING=false gemini -p "Reply OK"` → summarizer 비활성
3. 대용량 Read 후 summarizer 작동 확인 (실 사용 중 자동)

**Pass criteria**: 4 strategy 모두 toggleable + 활성 시 정상 동작.

## Appendix A — Troubleshooting

### A.1 "bkit Vibecoding Kit" 헤더 출력 안 됨
- `gemini extensions list` 실행하여 bkit 등록 확인
- `gemini extensions install <path>` 재설치
- `GEMINI_CLI_TRUST_WORKSPACE=true` 환경 변수 확인 (v0.39.1+ 필수)

### A.2 list_agents에 21 agent 안 나옴
- bkit MCP 서버 등록 확인: `cat .gemini/config.json | grep bkit`
- `gemini mcp list`로 bkit-server 활성 상태 확인

### A.3 /pdca plan 명령이 인식 안 됨
- `gemini commands list` 실행하여 pdca command 확인
- bkit-extension.json의 commands 필드 누락 가능성

### A.4 trust score < 60 으로 L4 진입 불가
- `.bkit/state/control-state.json` 확인
- Manual bootstrap: 사용자 결재로 trustScore 직접 60 이상 set (보안 의미상 신중)

### A.5 audit log 안 쌓임
- `.bkit/state/audit/<YYYY-MM-DD>/decisions.jsonl` 경로 확인
- L0~L2에서는 audit 일부만 기록 (L3+ 시 full)

### A.6 fixture miss in stub mode
- `BKIT_TEST_MODE=record` 로 한 번 실행하여 fixture 자동 생성 후 stub 재실행
