## STEP 0 — 시작 전 환경 체크 (소요 5분)


- jq (macOS): `brew install jq`
- jq (Linux): `apt install jq` 또는 `yum install jq`


**기대 출력 (둘 중 하나)**:


## STEP 4 — 자주 발생하는 문제 (트러블슈팅)

### 4.1 "bkit Vibecoding Kit ... activated" 메시지가 안 나옴

**원인 후보**:
1. 심볼릭 링크 미설정 → STEP 0.2 확인
2. `gemini-extension.json`에 mcpServers 정의 오류 → `cat gemini-extension.json | jq '.mcpServers'` 확인
3. Node.js 버전 미지원 (Node 16 이하) → Node 18+ 업그레이드

### 4.2 "code-analyzer agent was not found" (실제로 자주 보임)

**이건 정상입니다**. R-extra-1 사전 부채 때문에 `generalist` fallback이 실행됩니다. fallback 응답이 정상으로 출력되면 PASS입니다.

`v2.1.0-agent-dispatch-fix` sprint 완료 후에는 `code-analyzer`가 직접 실행됩니다.

### 4.3 "You have exhausted your capacity on this model. Your quota will reset after 1s..."

**이건 정상입니다**. Gemini 모델의 throttling 메시지. 1초~수초 대기 후 자동 retry됩니다. 대기 후 정상 응답이 나오면 PASS.

지속적으로 quota 초과 시: `gemini auth` 또는 Google AI Studio에서 API quota 확인.

### 4.4 "FatalUntrustedWorkspaceError"

**원인**: Gemini CLI v0.39.1+에서 untrusted workspace는 headless mode 차단.

**해결**:
```bash
# 옵션 A (권장): bkit이 자동으로 GEMINI_CLI_TRUST_WORKSPACE='true' 전파
# - 정상 동작이라면 별도 조치 불필요

# 옵션 B: 수동 trust
gemini auth trust
# 또는 --skip-trust 플래그 사용
gemini -p "..." --skip-trust
```

bkit의 `mcp/bkit-server.js`는 자식 gemini process에 `GEMINI_CLI_TRUST_WORKSPACE='true'`를 무조건 전파합니다 (v0.39.1+ 호환성). tc115 8/8 PASS이면 이 메커니즘은 정상.

### 4.5 "Ripgrep is not available. Falling back to GrepTool."

**이건 정보 메시지일 뿐**. 동작에는 영향 없습니다. 

성능 개선을 원하면:
- macOS: `brew install ripgrep`
- Linux: `apt install ripgrep` 또는 `yum install ripgrep`

### 4.6 baseline Pass < 1925

**원인 후보**:
1. bkit v2.0.7 PR이 잘못 머지됨
2. `.gemini/settings.json`이 손상됨
3. `lib/gemini/version.js` 또는 `tests/suites/tc38-feature-flags-matrix.js`가 잘못 변경됨

**대응**:
```bash
git diff main -- lib/gemini/version.js tests/suites/tc38-feature-flags-matrix.js bkit.config.json .gemini/settings.json
git log --oneline -5
```

**보고 필요**: `docs/04-report/sprints/v0.42.0-stable-migration-regression-<YYYYMMDD>.md` 생성 + `pdca-iterator` 호출.

### 4.7 npx v0.42.0 다운로드가 너무 느림

**원인**: 첫 호출 시 `~/.npm/_npx/` 캐시에 패키지 다운로드 (수 MB).

**해결**: 첫 호출만 느리고, 이후 호출은 캐시되어 빠릅니다. timeout이 필요하면:
```bash
# macOS: gtimeout (brew install coreutils)
# Linux: timeout
gtimeout 180 npx --yes @google/gemini-cli@0.42.0 -p "..."
```
