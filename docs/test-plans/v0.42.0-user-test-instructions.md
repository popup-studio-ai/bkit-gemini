# bkit v2.0.7 사용자 직접 검증 가이드 (Gemini CLI v0.42.0 stable + v2.0.7-upgrade ENH)

> **이 문서가 누구를 위한 것인가**: bkit Vibecoding Kit v2.0.7을 Gemini CLI v0.42.0 stable 환경에서 직접 동작 검증하려는 사용자 (개발자 / QA / 사용자 본인).
> **소요 시간**: Quick smoke ~5분 / Full validation ~30~60분 (LLM 호출 시간 포함) / ENH 카나리아 ~10분

## STEP 5 — 자주 발생하는 문제 (트러블슈팅)

### 5.1 "bkit Vibecoding Kit ... activated" 메시지가 안 나옴

**원인 후보**:
1. 심볼릭 링크 미설정 → STEP 0.2 확인

3. Node.js 버전 미지원 (Node 16 이하) → Node 18+ 업그레이드

### 5.2 "code-analyzer agent was not found" (실제로 자주 보임)

**이건 정상입니다**. R-extra-1 사전 부채 때문에 `generalist` fallback이 실행됩니다. fallback 응답이 정상으로 출력되면 PASS입니다.

`v2.1.0-agent-dispatch-fix` sprint 완료 후에는 `code-analyzer`가 직접 실행됩니다.

### 5.3 "You have exhausted your capacity on this model. Your quota will reset after 1s..."

**이건 정상입니다**. Gemini 모델의 throttling 메시지. 1초~수초 대기 후 자동 retry됩니다. 대기 후 정상 응답이 나오면 PASS.

지속적으로 quota 초과 시: `gemini auth` 또는 Google AI Studio에서 API quota 확인.

### 5.4 "FatalUntrustedWorkspaceError"

**원인**: Gemini CLI v0.39.1+에서 untrusted workspace는 headless mode 차단.

**해결**:
```bash
# 옵션 A (권장): bkit이 자동으로 GEMINI_CLI_TRUST_WORKSPACE='true' 전파
# - 정상 동작이라면 별도 조치 불필요

# 옵션 B: 환경 변수 직접 설정
GEMINI_CLI_TRUST_WORKSPACE=true gemini -p "..."

# 옵션 C: 수동 trust 또는 --skip-trust 플래그
gemini -p "..." --skip-trust
```

bkit의 `mcp/bkit-server.js`는 자식 gemini process에 `GEMINI_CLI_TRUST_WORKSPACE='true'`를 무조건 전파합니다 (v0.39.1+ 호환성). tc115 8/8 PASS이면 이 메커니즘은 정상.

### 5.5 "Ripgrep is not available. Falling back to GrepTool."

**이건 정보 메시지일 뿐**. 동작에는 영향 없습니다.

성능 개선을 원하면:
- macOS: `brew install ripgrep`
- Linux: `apt install ripgrep` 또는 `yum install ripgrep`
