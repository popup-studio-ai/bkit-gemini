```bash

# 기대 출력: bkit -> /path/to/bkit-gemini

# 또는 정식 install (사용자용)
gemini extensions install popup-studio-ai/bkit-gemini
gemini extensions list | grep bkit
```

미설치 또는 미링크 시: `gemini -p "anything"` 실행 시 `bkit Vibecoding Kit v2.0.7 activated` 메시지가 출력되지 않음 (이게 미설치 신호).

## §1. 테스트 매트릭스 (L1 ~ L5)

bkit의 테스트는 5개 레벨로 계층화된다:

| Level | 종류 | 도구 | 시간 | 목적 |
|---|---|---|---|---|
| **L1** | Unit | `node tests/run-all.js --suite <tcXX>` | 1초 ~ 1분 | 단일 모듈/함수 검증 |
| **L2** | Baseline (full regression) | `node tests/run-all.js` | 2~5분 | 전체 2046 테스트 회귀 검증 |
| **L3** | Spot (CLI invocation) | `gemini -p "..."` 직접 호출 | 30초 ~ 2분 | 실제 Gemini CLI invocation + bkit hook 동작 |
| **L4** | Agent catalog & dispatch | `gemini agents list` + `gemini -p "Use the <agent>..."` | 1~3분 | bkit agent 21개 인식 + dispatch (R-extra-1 검증) |
| **L5** | E2E canary scenario | 시나리오 A/B1/B2/C 4건 | 10~30분 | sprint 핵심 보존 카나리아 |

### 1.1 dual-version 정책

모든 L3/L4/L5 테스트는 **두 환경에서 동일하게 실행**:

```bash
# Environment 1: local installed (v0.39.1 또는 사용자 환경)
gemini ...

# Environment 2: v0.42.0 (npx 강제)
npx --yes @google/gemini-cli@0.42.0 ...
```

두 환경 결과가 일치해야 PASS. 차이 발생 시 §5 회귀 대응 절차로 이동.
