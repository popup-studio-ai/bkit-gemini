## 3. Cluster B — SEC-08 modes array regex (1건)

### 3.1 사실 매트릭스

테스트 (tests/suites/tc80-security-v200.js:881):
```js
const modesMatches = content.match(/modes\s*=\s*["plan"]/g);
assert(modesMatches && modesMatches.length >= 3, ...);
```

**버그**: `["plan"]`는 character class — 따옴표 `"` 한 글자 매치만 가능. `\[` `\]` escape 누락.

실제 `.gemini/policies/bkit-starter-policy.toml`은 `modes = ["plan"]` 형태가 정확히 3번 출현.

### 3.2 근본 대응

테스트 regex를 `/modes\s*=\s*\["plan"\]/g`로 수정. 회귀 testcase 추가하여 starter/dynamic/enterprise policy 각각 modes array 형태가 정확한지 동적 검증.

## 4. Cluster C — SEC-10 path/content expose (4건)

### 4.1 사실 매트릭스

`mcp/bkit-server.js`는 v2.0.x에서 `_textResponse()` helper로 리팩토링:

```js
// 현재 (정확함)
handleGetAgentInfo(args) {
  ...
  return this._textResponse({
    name: agent_name,
    file: agentInfo.file,           // ✓ 상대 파일명만
    description: agentInfo.description,
    recommendedModel: agentInfo.recommendedModel,
    safetyTier: agentInfo.safetyTier,
    exists: fs.existsSync(agentPath)  // agentPath는 응답에 포함 안 됨
  });
}

handleListAgents() {
  const agents = Object.entries(AGENTS).map(([name, info]) => ({
    name, description: info.description, recommendedModel: info.recommendedModel
  }));
  return this._textResponse({ agents });  // ✓ file/path 노출 없음
}
```

**테스트 패턴은 outdated**:
- TC80-95 / TC91-92: `return\s*\{\s*content` 또는 `JSON.stringify(\{` 직접 객체 리터럴 기대 → 현재는 `_textResponse(...)` helper
- TC91-89: `src.indexOf('handleGetAgentInfo(args)')` → dispatch case (line 615)에 우선 매치하여 잘못된 1200-char window
- TC91-94: `handleListAgents\(\)\s*\{[\s\S]*?return\s*\{` → `return this._textResponse({` 매치 안 됨, lazy quantifier가 다음 함수까지 흡수

### 4.2 근본 대응

**정적 grep 테스트 → 동적 실행 테스트로 강화** (보안 회귀를 진짜로 잡는 testcase):

1. 정적 grep 테스트는 helper-aware로 update:
   - `handleGetAgentInfo` 본문 범위를 함수 정의(`handleGetAgentInfo(args) {`)부터 매치하도록 정정
   - `_textResponse` 호출 인자 객체에 `agentContent`/`extensionPath`/`agentPath` 키가 없는지 검사

2. **신규 동적 회귀 testcase**: `tests/regression/security/mcp-response-leak.test.js`
   - BkitServer 인스턴스 생성 → `handleGetAgentInfo({agent_name: 'code-analyzer'})` 실제 호출
   - 반환 JSON 문자열에 file body 첫 100 chars 부분문자열 / 절대경로(`/Users`, `C:\\`, `PLUGIN_ROOT`) 노출 없음 검증
   - `handleListAgents()` 반환 문자열에도 동일 검증
   - error path: 잘못된 agent_name 전달 → 응답에 `extensionPath` 누설 없음 검증

이러면 미래에 누군가 실수로 `agentContent`나 `agentPath`를 응답에 추가해도 즉시 fail.
