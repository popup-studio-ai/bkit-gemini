## 1. Problem

Anthropic 2025-09 발표 Context Engineering 4-strategy가 bkit에 완전 통합되지 않은 상태.
v2.0.7-upgrade ENH-3은 mapping 분석 문서만 작성, **실제 통합 0건**.

| Strategy | bkit 현재 | Gap |
|----------|-----------|-----|
| 1. Compaction | `pre-compress.js`(97L)가 PDCA snapshot 저장만, 실 압축 없음 | trigger + JSON 추출 + restore 없음 |
| 2. Tool-result clearing | `after-tool.js`(244L)는 audit + L4 down-grade 처리, Read 결과 요약 없음 | result_size > N 자동 요약 미구현 |
| 3. Memory tool | `agent-memory.js`(214L) AgentMemoryManager 격리 file-based, Gemini Auto Memory inbox 양방향 sync 0건 | extraction.patch 매핑 미구현 |
| 4. Sub-agent isolation | `context-fork.js`(535L)에 forkContext/mergeContext 존재, S3 dispatch와 미연결 | spawn된 agent context fork 자동화 X |

사용자 영향: 긴 PDCA sprint(2~5h) 시 토큰 누적 → 응답 지연 + 컨텍스트 오염 → hallucination 가능성.

## 2. Architecture (4-wave sequential)

### Wave 1 — Compaction
```
hooks/scripts/pre-compress.js (확장)
    ↓ threshold check (turns/tokens)
lib/core/compactor.js (NEW)
    ├─ extractDecisions(history) — 결정/미해결/핵심 코드 위치 JSON
    ├─ shouldCompact(state) — trigger 판정
    └─ saveSnapshot() — .bkit/snapshots/compact-<ts>.json
    
환경 변수: BKIT_COMPACTION=false → skip
```

### Wave 2 — Tool-result clearing
```
hooks/scripts/after-tool.js (확장)
    ↓ tool=Read 또는 Glob/Grep 등
lib/core/tool-result-summarizer.js (NEW)
    ├─ shouldSummarize(result, threshold) — 크기 기준
    ├─ summarize(result) — "first N lines + file:line ref + size"
    └─ preserveRefs() — 미래 lookup 가능한 reference 유지
```

### Wave 3 — Memory tool 통합
```
lib/core/agent-memory.js (확장)
    ├─ exportAsExtractionPatch() — Gemini extraction.patch 스키마 변환
    └─ importFromExtractionPatch(patch) — 양방향 sync
스키마: { agent, scope, patches: [{ session, summary, findings, ts }] }
```

### Wave 4 — Sub-agent isolation
```
lib/gemini/agent-dispatch.js (확장) + lib/gemini/context-fork.js (활용)
    ↓ spawn 직전
forkContext() → 격리 fork
    ↓ spawn agent
agent 실행 (main context X)
    ↓ 종료
mergeContext(forkResult) → main에 결과만 반환
```

## 5. Risks

| Risk | Mitigation |
|------|-----------|
| Compaction이 사용자 컨텍스트 손실 | snapshot은 무손실 JSON (decisions + open + code refs). BKIT_COMPACTION=false 토글. |
| Tool-result clearing이 중요 정보 누락 | 요약 + reference (file:line) 보존. 원본 raw는 disk에 unchanged. |
| Memory schema migration이 기존 21 file 손상 | importFromExtractionPatch는 incremental merge, 원 file 무수정. 동적 회귀 무손상 검증. |
| Sub-agent isolation이 S3 dispatch 깨뜨림 | 의존성: S3 Track A 정상 동작 확인. AC-T6 + agent-dispatch regression 동시 검증. |
