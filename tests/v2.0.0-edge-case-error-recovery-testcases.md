## Key Observations from Code Analysis

1. **version.js** has robust null/injection guards but `parseVersion` silently returns v0.29.0 defaults for any invalid input -- callers must be aware this is a "safe but potentially misleading" default.

2. **context-fork.js** has a named-snapshot vs forkId mismatch: `forkContext` stores as `{name}.json` but `mergeForkedContext` looks for `fork-{forkId}.json`. If `options.name` is provided, the merge path will not find the file. This is a potential bug (TC J1-03).

3. **session-start.js** references `getAdapter()` as a function import from `platform.js`, but `platform.js` exports a singleton instance (`module.exports = new GeminiAdapter()`), not a `getAdapter` function. This suggests either `platform.js` was refactored or `session-start.js` uses a different import path through `lib/common.js`. This mismatch should be verified (relates to TC G1-01).

4. **policy.js** `validateTomlStructure` correctly catches the v0.32.0+ `toolName` vs `toolname` case-sensitivity breaking change.

5. All hook scripts follow fail-open pattern: errors result in `{ status: 'allow' }` or `process.exit(0)`, never blocking the session.
