## Executive Summary Output Rule

**When to output**: After completing PDCA document work (/pdca plan, /pdca design, /pdca report, /plan-plus)

**What to output**: Extract and display the Executive Summary section from the created/updated document:
1. Project overview table (Feature, dates, scope)
2. Results summary (Match Rate, items, files, lines if applicable)
3. Value Delivered 4-perspective table (Problem / Solution / Function UX Effect / Core Value)

**Position**: Output Executive Summary BEFORE the bkit Feature Usage report.
Both are required: Executive Summary (after document work) + Feature Usage (always).

**Why**: Users should see the summary immediately without opening the file.
